Исправить валидацию географии в форме заявки и других адресных формах.

Проблема:
Пользователь может ввести несуществующий район и несуществующий населённый пункт вручную. Например:
- район: "тупринский"
- населённый пункт: "томтор"
и система принимает текст, даже если он не выбран из подсказки.

Требования:

1. Поля "Район" и "Населённый пункт" должны принимать только значения из подсказок.
   Простого ручного ввода недостаточно.

2. Если пользователь ввёл текст, но не выбрал вариант из списка:
   - показывать ошибку под полем:
     "Выберите значение из списка"
   - не разрешать отправку формы.

3. Если район не найден в справочнике:
   - показывать:
     "Район не найден. Проверьте название или выберите другой регион."
   - не разрешать отправку формы.

4. Если населённый пункт не найден:
   - показывать:
     "Населённый пункт не найден. Проверьте название или выберите район/регион."
   - не разрешать отправку формы.

5. При смене региона:
   - очищать выбранный район;
   - очищать выбранный населённый пункт;
   - очищать адрес объекта.

6. При смене района:
   - очищать выбранный населённый пункт;
   - очищать адрес объекта.

7. Адрес объекта может быть ручным, но если пользователь выбирает подсказку — сохранять полный адрес.

8. Исправить все формы, где есть район и населённый пункт:
   - создание заявки;
   - профиль инженера / территория обслуживания;
   - фильтры заявок, если там есть район/населённый пункт.

9. Backend-защита:
   - при создании заявки backend должен дополнительно проверять, что район и населённый пункт существуют в справочнике для выбранного региона.
   - если не существуют — возвращать HTTP 400:
     {
       "error": "Некорректный район или населённый пункт"
     }

10. Выполнить typecheck frontend/backend.

11. Проверить:
   - Республика Саха (Якутия) + "тупринский" → ошибка, заявка не создаётся.
   - Республика Саха (Якутия) + "Алданский" → принимается.
   - Алданский район + "Алдан" → принимается.
   - Алданский район + "несуществующий город" → ошибка.
